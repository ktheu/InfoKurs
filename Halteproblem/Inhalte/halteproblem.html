Das Halteproblem

Das Halteproblem ist die Frage, ob ein Algorithmus zu einem Ende gelangt.
Das Halteproblem ist algorithmisch nicht entscheidbar, d.h. es gibt keinen Algorithmus, der dies für alle Algorithmen und beliebige Eingaben entscheiden kann. 

def haltetest(pgm,inp):
  # returned True, wenn pgm angesetzt auf inp hält
  # sonst False

def quer(x):
  if haltetest(x,x):
    while True: pass

Annahme: es gäbe eine eine Funktion haltetest, die das Halteproblem löst. Die Frage, ob quer(quer) hält, führt dann zum Widerspruch.

Das Postsche Korrespondenzproblem (PKP)

Gegeben sind endliche viele Typen von Dominosteinen auf denen oben und unten ein Wort (z.B. 01-Folge) aufgedruckt ist. Von jedem Typ stehen beliebig viele Steine zur Verfügung. Kann man eine Folge von Dominosteine finden, so dass sich oben und unten das gleiche Gesamtwort ergibt?

Beispiele

Welche Reihenfolge löst das PKP-Problem? 

 1: (1,101), 2: (10,00), 3: (011,11)

Mit der Folge 1,3,2,3 ergibt sich oben und unten: 101110011

Welche Reihenfolge löst das PKP-Problem? 

 1: (001,0), 2: (01,011), 3: (01,101), 4: (10,001)

Eine kürzeste Lösung besteht schon aus 66 Paaren:
2,4,3,4,4,2,1,2,4,3,4,3,4,4,3,4,4,2,1,4,4,2,1,3,4,1,1,3,4,4,4,2,1,2,1,1,1,3,4,3,4,1,2,1,4,4,2,1,4,1,1,3,4,1,1,3,1,1,3,1,2,1,4,1,1,3

Unentscheidbarkeit von PKP: Es gibt keinen Algorithmus, der für beliebige PKP-Instanzen entscheiden kann, ob es eine Lösung gibt.

Wie viele Probleme sind algorithmisch nicht lösbar?

Jeder Algorithmus ist eine endliche Folge von Zeichen, die als binäre Zahl interpretiert werden kann. d.h. es gibt höchstens so viele Algorithmen wie es natürliche Zahlen gibt: abzählbar unendlich viele. (Mächtigkeit von Mengen)

Betrachte die Menge der Entscheidungsprobleme, die jeder natürlichen Zahl entweder 1 oder 0 zuordnet (JA/NEIN). Davon gibt es so viele, wie es Teilmengen von natürlichen Zahlen gibt, also überabzählbar unendlich viele.

Die Menge der Algorithmen ist so groß wie \(\mathbb{N}\). Die Menge der Entscheidungsprobleme ist so groß wie \(\mathbb{R}\).

Also: Für fast alle Entscheidungsprobleme gibt es keine Algorithmen.
