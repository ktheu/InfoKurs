Variablen, Typen, arithmetische und boolesche Operatoren

Aufgaben     Lösungen     Videos    

Eingaben in der Shell werden sofort ausgewertet.

>>> 8
8
>>> 3.2
3.2
>>> 'Hello'
'Hello'
>>> True
True
>>> None         # wertet sich zu nichts aus
>>>

Jedes Objekt hat einen Typ. 

>>> type(8)
<class 'int'>
>>> type(3.2)
<class 'float'>
>>> type('Hello')
<class 'str'>
>>> type(True)
<class 'bool'>
>>> type(None)
<class 'NoneType'>

Arithmetische Operatoren 

>>> 11+3,11*3,11/3
(14, 33, 3.6666666666666665)
>>> 11//3      # ganzzahlige Division = Division ohne Rest
3
>>> 11%3       # 11 modulo 3 = Rest der Division 11 durch 3
2
>>> 2**3       # Exponentiation: 2 hoch 3
8
>>>

Auswertungsreihenfolge bei arithmetischen Ausdrücken: Klammern, **, {* / // %},{+ -}, bei gleichwertigen Operatoren von links nach rechts, außer Exponentiation, die wird ohne Klammerung von rechts nach links ausgewertet. 

>>> 3*(4+1)-7
8
>>> 4+2*3**2/4
8.5
>>> 8%2*3
0
>>> 8*2%3
1
>>> 2**3**2
512
>>> (2**3)**2
64
>>>

(Quiz 1)

Mit einer Zuweisung wird einer Variablen ein Wert zugewiesen. 

>>> pi = 3.1415
>>> radius = 2.2
>>> flaeche = pi*(radius**2)
>>> flaeche
15.204860000000004

Wir können den Wert von Variablen durch erneute Zuweisung verändern. 

>>> x = 5
>>> x = x + 1
>>> x
6

Kurzformen (erweiterte Zuweisung): 

x += 1    entspricht  x = x + 1
x -= 1    entspricht  x = x - 1
x *= 2    entspricht  x = x * 2 ... usw.

Variablennamen sind case-sensitiv. Nach mindestens einem führenden Buchstaben oder Unterstrich kann so ziemlich jedes Zeichen folgen. Wir schränken uns freiwillig ein und nehmen in der Regel nur Unterstrich, die Buchstaben des englischen Alphabets und die Ziffern. Wenn es der Lesbarkeit des Programms nicht schadet, erlauben wir uns kurze Variablennamen. 

>>> i = 0
>>> s = 'Habicht'
>>> b = True
>>> x = 2.5

Ansonsten nehmen wir sprechende Namen, beginnend mit einem kleinen Buchstaben. Zusammengesetze Namen entweder in camelCase oder mit Unterstrich verbunden. 
>>> zaehl = 1
>>> radius = 2.2
>>> anzahlTreffer = 5            # camelCase
>>> entfernungZurWand = 3.2
>>> anzahl_treffer = 5           # mit Unterstrich
>>> entfernung_zur_wand = 3.2

Die Bedeutung eines Operators hängt von den Typen der beteiligten Objekte ab. 

>>> 3 + 4
7
>>> '3' + '4'
'34'
>>> 3 * 4
12
>>> '3' * 4
'3333'
>>> '3' * '4'
Traceback (most recent call last):
  ...
TypeError: can't multiply sequence by non-int of type 'str'

Wenn es ein sinnvolles Ergebnis gibt, können unterschiedliche Typen ineinander umgewandelt werden (casting). 

>>> float(4)
4.0
>>> str(4)
'4'
>>> bool(4)      # alles ungleich 0 ergibt True
True
>>> int(3.2)
3
>>> str(3.2)
'3.2'
>>> bool(3.2)
True
>>> int('5')
5
>>> float('5')
5.0
>>> bool('5')
True
>>> bool('')     # nur der leere String ergibt False
False
>>> int('5.2')
Traceback (most recent call last):
  ...
ValueError: invalid literal for int() with base 10: '5.2'

Vergleichsoperatoren für ints und floats. 

>>> i = 5
>>> j = 7
>>> i>j, i>=j, i<j, i<=j, i==j, i!=j
(False, False, True, True, False, True)

Logische Operatoren für booleans. 

>>> a = True
>>> b = False
>>> a and b      # True wenn beide True
False
>>> a or b       # True wenn mindestens einer True
True
>>> not a        # True wenn a False
False

Ein boolescher Ausdruck wertet sich immer zu True oder False aus 

>>> a = 2
>>> b = 3
>>> (a*b > a+b) and (a % 2 == 0)
True

Mehrfachanweisungen und Vertauschung

>>> x,y = 2,7
>>> x
2
>>> y
7
>>> x,y
(2, 7)
>>> x,y = y,x
>>> x,y
(7, 2)
